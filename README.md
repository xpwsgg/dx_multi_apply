# BatchApply — 批量入场申请工具

公司园区访客管理系统每次只能提交一天的入场申请，本工具支持一次性提交多天、多访客、多接待人的批量申请，大幅减少重复操作。

## 功能特性

### 登录与认证

- **短信验证码登录** — 输入手机号获取验证码完成登录，全程实时推送登录进度
- **手动 Token 导入** — 支持从浏览器粘贴 `acToken` 跳过短信流程，后端自动校验格式（64 位十六进制）
- **自动恢复登录** — 启动时自动读取并校验本地保存的 Token；服务端异常时保留登录状态，Token 失效时自动清除
- **切换账号** — 一键清除当前登录态并切换新账号

### 访客管理

- **逐条 / 批量添加** — 支持单条输入或弹窗中按行批量导入身份证号，自动去重，实时显示查询进度
- **信息自动带出** — 输入身份证号后自动查询访客姓名、电话、免冠照片、身份证照片、社保/在职证明，提交时一并带入申请单
- **预约记录查询** — 按身份证号查询历史预约记录，展示单号、访客/接待人联系方式、权限时间段、权限状态（审核中/已同意/已生效/已失效等）、申请时间

### 接待人管理

- **逐条 / 批量添加** — 输入工号或批量导入查询接待人姓名、部门、联系方式
- **多接待人支持** — 按”日期 × 接待人”笛卡尔积生成独立任务，每个接待人独立提交

### 日期选择

- **自研日历组件** — 月份导航、当日高亮、已选高亮、禁用过去日期，点击外部或 Escape 关闭
- **快捷按钮** — `明天`、`未来 3 天`、`未来 5 天`、`未来 10 天`，均从明天起算
- **范围限制** — 最大跨度 10 天，超出自动截断并提示

### 任务清单

- **自动生成** — 根据”日期 × 接待人”生成待执行任务表格，含日期、星期、接待人、部门、状态
- **本地去重** — 自动识别已提交记录并标记为”已存在”，支持一键批量移除已存在任务
- **预检汇总** — 实时展示待提交数、跳过数、预计等待时长
- **灵活编辑** — 支持删除单条、批量删除已存在项、恢复全部任务，支持”仅看待提交”过滤视图

### 批量提交

- **串行执行** — 任务按顺序逐条发送，请求间随机等待 30–50 秒，降低限流风险
- **智能等待** — 若后续全为已跳过任务，自动省略等待时间
- **强制重提确认** — 任务包含本地已存在记录时，弹出原生确认对话框
- **实时反馈** — 每条任务状态实时更新（待提交 → 提交中 → 成功/跳过/失败/已停止），表格自动滚动到当前行
- **当前任务卡片** — 右侧面板显示当前任务、下一条任务及剩余数量
- **进度与倒计时** — 显示 `已处理 / 总数` 进度和等待倒计时

### 失败处理与重试

- **完成统计** — 提交完成后弹出汇总，显示成功/跳过/失败各自数量
- **复制失败项** — 将失败任务格式化复制到剪贴板
- **一键重试** — 仅保留失败项并重置为待提交状态，直接重试

### 日志系统

- **前端实时日志** — 所有查询和提交结果实时写入日志面板（最近 200 条），支持查看原始响应、复制全部、清空
- **后端磁盘日志** — JSONL 格式写入 `app.log`，文件上限 512KB 自动精简，启动时清空

### 安全与防误操作

- **手动停止** — 支持随时停止正在执行的批量任务，原生确认对话框二次确认
- **关窗保护** — 任务执行中关闭窗口弹出确认框
- **并发保护** — 防止重复提交（原子锁检测）
- **清除表单** — 确认后清空所有数据并同步清除登录状态

### 数据持久化

- **表单自动保存** — 手机号、访客身份证号、接待人工号自动保存（500ms 防抖），下次启动自动恢复并重新查询
- **提交记录数据库** — SQLite 存储历史提交记录，支持去重检测和 schema 自动迁移
- **会话恢复提示** — 启动时显示恢复进度 Banner

## 使用流程

1. 输入申请人手机号，通过短信验证码或手动导入 `acToken` 登录
2. 录入访客身份证号并查询，可逐条添加或”批量添加访客”一次导入多条
3. 录入接待人工号并查询，可逐条添加或”批量添加接待人”一次导入多条
4. 选择日期范围，或使用 `明天` / `未来 3 天` / `未来 5 天` / `未来 10 天` 快捷按钮
5. 系统自动生成”日期 × 接待人”任务清单；可删除单条、批量移除已存在项，或切换”仅看待提交”视图
6. 点击”开始提交”；如包含已提交过的任务，系统弹窗确认是否强制重提
7. 在实时面板查看提交进度、等待倒计时、当前/下一任务卡片
8. 提交完成后查看统计；如有失败项，可”复制失败项”或”仅保留失败项重试”

## 提交规则

- **成功判定**：服务端返回 `HTTP 2xx` 且响应体 `success === true`
- **去重规则**：按”同一接待人 + 同一访客组合 + 同一日期”识别已提交任务，提交时自动跳过
- **强制重提**：待执行列表包含已存在任务时弹窗确认，确认后强制重新提交
- **等待策略**：串行执行，请求间随机等待 30–50 秒；后续全为跳过项时自动省略等待
- **日期限制**：开始日期不早于当天；快捷日期从明天起算；区间最长 10 天，超出自动截断
- **失败恢复**：部分失败不丢失上下文，可直接保留失败项重试
