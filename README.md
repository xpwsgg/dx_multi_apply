# 批量申请工具（Tauri + React）

这是一个用于批量申请提交的桌面工具。

## 功能概览

- 选择开始日期和结束日期，生成闭区间日期列表（包含首尾）。
- 在提交前可删除不需要的日期。
- 调用 Tauri 命令串行提交，支持手动停止。
- 日志实时展示每条提交结果，并保留服务端原始响应 `responseRaw`。
- 提交间隔提供前端倒计时显示，关闭软件和未完成停止任务会二次确认。
- 本地保存最近 30 天申请历史，新增申请会提示重复日期并在提交时自动跳过。

## 本地开发

```bash
pnpm install
pnpm tauri dev
```

## 页面使用流程

1. 选择开始日期与结束日期。
2. 点击“生成日期列表”。
3. 在“待提交日期”中删除不需要的日期。
4. 点击“开始提交”执行批量提交。
5. 运行中如需终止，点击“停止提交”。

## 提交规则

后端会按日期升序串行提交，并在每次成功后随机等待 `60..=120` 秒。
对于本地历史中已存在的日期，后端会标记为 `skipped` 并跳过请求。

任一日期失败即停止后续提交，失败判定为以下任一条件成立：

- HTTP 状态码不是 2xx。
- 响应体不是可解析 JSON。
- JSON 中 `success !== true`。

## 日志字段说明

`batch-log` 事件包含以下常见字段：

- `date`: 当前处理日期。
- `result`: `success` / `failed` / `stopped`。
- `waitSeconds`: 成功后等待秒数。
- `reason`: 失败或停止原因。
- `responseRaw`: 服务端原始响应字符串（用于排查问题）。

## 国际化

- 前端界面使用中文。
- 后端（Rust）所有错误信息均为中文，便于调试和问题定位。
