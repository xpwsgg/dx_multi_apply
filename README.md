# BatchApply — 批量入场申请工具

公司园区访客管理系统每次只能提交一天的入场申请，本工具支持一次性提交多天、多访客、多接待人的批量申请，大幅减少重复操作。

## 功能特性

- **短信验证码登录 / Token 导入** — 支持短信验证码登录，也支持手动粘贴 `acToken`；登录状态会本地保存并在启动时自动恢复
- **访客信息自动带出附件** — 输入身份证号后自动查询访客姓名、电话、免冠照片、身份证照片，以及社保/在职证明，并在提交时一并带入申请单
- **接待人信息查询** — 输入工号自动获取接待人姓名、部门、联系方式，支持多接待人独立提交
- **批量添加访客与接待人** — 支持逐条添加，也支持在弹窗中按行批量导入身份证号 / 工号；输入后可直接回车确认
- **日期快捷选择** — 支持手动选择开始/结束日期，也支持 `明天`、`未来 3 天`、`未来 5 天`、`未来 10 天` 快捷按钮
- **任务清单可编辑** — 自动按“日期 × 接待人”生成待执行任务，提交前可删除单条任务，灵活调整本次执行范围
- **重复任务处理** — 自动识别本地已提交记录并标记为已存在；若本次任务中包含已存在项，提交前会弹窗确认是否强制重新提交
- **失败项重试** — 批量提交结束后可一键复制失败项，或仅保留失败项再次重试
- **实时日志与原始响应** — 所有查询和提交结果实时写入日志面板，支持查看服务端原始响应、复制全部日志和清空日志
- **预约记录查询** — 可按访客身份证号查询历史预约记录，展示单号、访客电话、接待人联系方式、权限生效/截止时间、权限状态、申请时间等关键信息
- **表单状态持久化** — 自动保存申请人手机号、访客身份证号、接待人工号，下次启动后自动恢复
- **安全防误操作** — 支持手动停止正在执行的任务；任务执行中关闭窗口会弹出确认框；清除表单会同步清除登录状态

## 使用流程

1. 输入申请人手机号，通过短信验证码或手动导入 `acToken` 登录
2. 录入访客身份证号并查询，可逐条添加，也可通过“批量添加访客”一次导入多条
3. 录入接待人工号并查询，可逐条添加，也可通过“批量添加接待人”一次导入多条
4. 选择到访日期范围，或直接使用 `明天` / `未来 3 天` / `未来 5 天` / `未来 10 天` 快捷按钮
5. 系统自动生成“日期 × 接待人”的任务清单；如有不需要执行的项，可在提交前删除单条任务
6. 点击“开始提交”执行批量提交；如果包含本地已提交过的任务，系统会先确认是否强制重新提交
7. 在日志弹窗查看实时进度、等待倒计时、失败原因和服务端原始响应
8. 如果存在失败项，可在完成弹窗中选择“复制失败项”或“仅保留失败项重试”

## 提交规则

- **成功判定**：以服务端返回 `HTTP 2xx` 且响应体中的 `success === true` 作为提交成功标准
- **去重规则**：默认按“同一接待人 + 同一日期”识别本地已提交任务，并在提交时自动跳过
- **强制重提**：当待执行列表包含已存在任务时，会弹窗确认；确认后才会强制重新提交这些任务
- **等待策略**：批量任务按顺序串行执行，请求之间会随机等待，降低接口限流风险
- **日期限制**：开始日期不能早于当天；快捷日期默认从明天开始计算；日期区间最长 10 天，超出会自动截断
- **失败恢复**：部分失败不会丢失上下文，可直接保留失败项重试

## 发布流程

1. 同步更新三处版本号：`package.json`、`src-tauri/tauri.conf.json`、`src-tauri/Cargo.toml`
2. 提交代码并创建 `vX.Y.Z` 格式的 tag
3. 推送到 GitHub，Actions 自动构建并创建 Release
4. 如需手动重跑发布流程，请对目标 tag 触发 workflow，而不是直接对 `main` 触发：

```bash
gh workflow run release.yml --ref vX.Y.Z
```
