# 需求说明

## 1. 项目背景

公司园区访客管理系统基于宜搭（阿里低代码平台）构建，员工需要为外部访客填写入场申请。原系统为 Web 页面，每次只能提交一天的申请，对于需要连续多天入场的访客，操作非常繁琐。

本工具（BatchApply / 批量入场申请）旨在自动化这一重复流程，支持一次性提交多天、多访客、多接待人的入场申请。

## 2. 原系统操作流程

1. 打开登录页：`https://iw68lh.aliwork.com/o/fk_login_app`
2. 输入手机号，获取短信验证码
3. 提交验证码完成登录，跳转至首页并将 `acToken` 写入 `localStorage`
4. 在首页填写访客申请单（每次仅限一天）
5. 查询预约记录

> 所有请求中的 `account` / `regPerson` 即申请人手机号。

## 3. 功能需求

### 3.1 登录认证

- 输入申请人手机号，通过 API 自动获取验证码并登录
- 登录成功后保存 `acToken` 到本地，供后续查询使用
- 支持 token 有效性检测，失效时提示重新登录
- 支持切换登录账号

### 3.2 访客信息管理

- 输入访客身份证号，自动查询并显示访客姓名、电话等信息
- 支持添加多个访客
- 查询结果缓存，避免重复请求

### 3.3 接待人信息管理

- 输入接待人工号，自动查询并显示姓名、部门、电话等信息
- 支持添加多个接待人
- 每个接待人独立提交申请（同一访客 + 同一日期 + 不同接待人 = 不同申请单）

### 3.4 批量日期提交

- 选择开始日期和结束日期，自动生成日期范围
- 显示已存在的申请日期，提交时自动跳过
- 按日期串行提交，每次提交间隔随机等待（防止接口限流）
- 支持手动停止正在执行的批量任务
- 提交结果实时显示在日志面板中

### 3.5 预约记录查询

- 根据访客身份证号查询已有的预约记录
- 显示完整记录信息：单号、访客姓名、电话、到访公司、园区、申请类型、接待人、生效/截止时间、权限状态、申请时间
- 权限状态映射为中文（审核中、审核拒绝、审核同意、审核通过权限未生效、权限已生效、权限已失效）
- 查询请求和响应记录到日志面板

### 3.6 日志系统

- 实时显示所有操作日志（提交、查询、跳过、失败等）
- 日志按倒序排列（最新在上）
- 支持复制全部日志、清空日志
- 日志同时持久化到本地文件

### 3.7 表单状态持久化

- 自动保存上次使用的申请人手机号、访客身份证号、接待人工号
- 下次启动时自动恢复表单状态

## 4. 非功能需求

### 4.1 跨平台支持

- macOS (Apple Silicon + Intel)
- Windows x64（同时提供安装版和免安装便携版）

### 4.2 安全性

- 不存储访客个人信息（仅存储身份证号用于表单恢复）
- `acToken` 存储在应用本地数据目录
- 无远程数据上报

### 4.3 易用性

- 强制浅色主题，保证各平台视觉一致
- 日期选择器中文本地化
- 操作按钮在任务执行期间禁用，防止误操作
- 关闭窗口时若有任务执行中则弹出确认对话框
